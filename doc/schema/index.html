The Root Schema - Represents the mapping from a DynamoDB table to an Exasol Virtual Schema table

The structure of this mapping definition follows the structure of the data in the DynamoDB. This object represents a DynamoDB table. Its properties are modeled in the children property of this object. Only properties of the DynamoDB table that shall be mapped are modeled.

Each of this objects must be stored in a single .json file an get uploaded to an directory in bucketfs.

toTableMapping: object

This mapping is used for normalizing documents from DynamoDB. It creates a new table in the resulting virtual schema named: <NAME_OF_PARENT>_<GIVEN_NAME>.

The columns of that new table are defined using the mapping property. If the parent table defined a key using one or more toString mappings for a DynamoDB primary key or secondary index this will be used as foreign key.

Supported DynamoDB types: List (L).

destTableName: string
mapping: mappingDefinition

Defines the mapping for the columns of the new table. If the DynamoDB list contains non objects like strings here directly a toString mapping can be applied. Otherwise, if the list contains objects, you can navigate through its properties using children.
