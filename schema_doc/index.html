The Root Schema - Represents the mapping from a DynamoDB table to an Exasol Virtual Schema table

The structure of this mapping definition follows the structure of the data in the DynamoDB. This object represents a DynamoDB table. Its properties are modeled in the fields property of this object. Only properties of the DynamoDB table that shall be mapped are modeled.

Each of this objects must be stored in a single .json file an get uploaded to an directory in bucketfs.

Properties:

NAME_OF_DYNAMODB_COLUMN: mappingDefinition

It must be specified a least one column.

mappingDefinition

key: string , x ∈ { local , global }

Setting this property marks a column as key.

Key columns are used as foreign keys from ToTableMappings for nested lists.

A key can either be local or global. A global key must be unique over all rows of the table. A local key must only be unique in the scope of the nested list.

mappingDefinition: object

A mapping definition can either be:

  • a mapping to a column or table
  • an step into the property tree of the DynamoDB table

toTableMapping: object

This mapping is used for normalizing documents from DynamoDB. It creates a new table in the resulting virtual schema named: <NAME_OF_PARENT>_<GIVEN_NAME>.

The columns of that new table are defined using the mapping property. If the parent table defined a key using one or more toString mappings for a DynamoDB primary key or secondary index this will be used as foreign key.

Supported DynamoDB types: List (L).

destTable: destTable
mapping: mappingDefinition

Defines the mapping for the columns of the new table. If the DynamoDB list contains non objects like strings here directly a toString mapping can be applied. Otherwise, if the list contains objects, you can navigate through its properties using fields.

description: description

required: boolean

If this property is set to true, queries will abort if the given mapping can't be found a single document. Else, the query will continue and set this cell to a default value. By default this is property is set to false.
